## 1. 背景与目标

### 1.1 为什么重构

诗云学习版（`诗云-学习版.html`）当前采用一套「水墨赛博 / Ink-Cyber」深色设计系统。代码可运行，但在真实使用中暴露出明显问题：

- **不够美观**：深色赛博风与「给孩子学古诗词」的产品定位错位，护眼性差。
- **组件不合理**：大量页面用内联 `style="..."` 拼装（如 `home.js` 首页几乎全是内联样式），缺乏统一组件，视觉不一致、难维护。
- **交互不友好**：英文技术标签（`SYSTEM ONLINE`、`Learned / Mastered / Queue`）不适合儿童；反馈、空状态、加载态缺失。
- **存在交互 Bug**：诗词详情页的「朗读 / 慢速 / 停止 / 收藏」在缺少音频数据时整条控件栏失效，且无降级提示（详见 §3）。

### 1.2 目标

1. 用一套**明亮国风**设计系统替换深色赛博风，护眼、契合古诗词气质、适合儿童。
2. 建立**统一组件库**，消灭散落的内联样式，所有 UI 走 class。
3. 修复全部已知交互 Bug，补齐加载 / 空 / 禁用 / 错误等状态。
4. 提供**样式合格验证**手段：组件样例页（Style Guide）+ 视觉走查清单 + 自动化检查。

### 1.3 设计决策（自主决策，待用户评审）

| 维度 | 决策 | 理由 |
|------|------|------|
| 目标用户 | 6–12 岁儿童为主，家长辅助 | 产品定位（部编版 1–6 年级 112 首） |
| 主力设备 | 平板 / 桌面大屏优先，移动端自适应 | 家庭学习场景，大屏更适合朗读跟读 |
| 视觉风格 | 明亮国风：宣纸底 + 水墨墨色 + 朱砂点缀 | 护眼、传统、童趣兼具 |
| 语言 | 全中文文案，去除英文技术黑话 | 儿童可读 |
| 重构范围 | 5 大模块（首页/学习/复习/考核/进度）+ 打印 全面重构 | 一次性统一体验 |

## 2. 设计原则

1. **内容为王**：诗词正文是绝对主角，排版要有呼吸感，字大、行宽、留白足。
2. **一眼可懂**：每个页面 3 秒内让孩子知道「现在能做什么」，大按钮、强对比、图标 + 文字双标注。
3. **即时反馈**：每次点击都有视觉/动效反馈；加载有骨架屏，空状态有引导，错误有友好提示。
4. **降级优雅**：AI 媒体（配图/音频）缺失时，用占位与提示替代，绝不出现「死按钮」。
5. **一致性**：颜色、间距、圆角、阴影、动效全部来自 Design Tokens，组件可复用。
6. **无障碍**：对比度达 WCAG AA，可键盘操作，`aria-*` 完整，尊重 `prefers-reduced-motion`。

## 3. 现状问题诊断（基于真实代码）

### 3.1 严重：详情页音频控件失效（用户反馈的核心 Bug）

**位置**：[src/js/ui/learn.js](../../../src/js/ui/learn.js) `renderPoemDetail`

```js
<button class="audio-btn audio-btn--play" id="audio-play" ${poem.audio ? '' : 'disabled'}>
<button class="audio-btn" id="audio-slow" ${poem.audio ? '' : 'disabled'}>
<button class="audio-btn" id="audio-stop" ${poem.audio ? '' : 'disabled'}>
...
if (poem.audio) {           // ← 缺音频时整段事件不绑定
  currentAudio = createAudio(poem.audio);
  ...
}
```

**根因**：当前构建嵌入的是**元数据**（`poems-meta.js`），AI 生成器尚未运行，所以每首诗的 `poem.audio` / `poem.image` / `poem.pinyin` 均为空。于是：

- 朗读 / 慢速 / 停止三个按钮被 `disabled`，点不动；
- 收藏按钮本身逻辑正确，但夹在三个死按钮中间，整条栏看起来都「坏了」。

**修复方向**（详见 §6.3）：
1. 无音频时用浏览器 **TTS（`speechSynthesis`）兜底朗读**，按钮始终可用；
2. 控件改为「播放/暂停」单一主按钮 + 速度切换 + 收藏，状态机清晰；
3. 缺图缺音时显示明确占位与「内容生成中」说明，而非禁用。

### 3.2 中：内联样式泛滥，无组件复用

**位置**：[src/js/ui/home.js](../../../src/js/ui/home.js)

首页几乎整页由内联 `style="..."` 拼装（数字字号、颜色、栅格都写死在 JS 字符串里）。问题：

- 视觉无法统一调整（改一个间距要改 N 处）；
- 同类卡片在不同页面长得不一样；
- 不利于主题切换与维护。

**修复方向**：所有内联样式抽取为 `main.css` 中的语义化 class（`.stat-card`、`.review-banner` 等），JS 只产出结构与数据。

### 3.3 中：文案与风格不适合儿童

- 英文黑话：`SYSTEM ONLINE · N poems`、`Learned / Mastered / Queue`、`format-classic`。
- 深色 `#0a0a0f` 背景 + 霓虹辉光，长时间阅读古诗词易疲劳。

**修复方向**：全中文化文案；浅色宣纸主题。

### 3.4 轻：状态缺失

- 列表搜索无结果时仅一行小字，无插画引导；
- 无骨架屏 / 加载态；
- 切换用户、收藏成功等操作缺 Toast 反馈。

## 4. 视觉设计系统（明亮国风）

### 4.1 色彩 Tokens

```css
:root {
  /* 宣纸底色（背景层次） */
  --paper-0: #faf6ee;   /* 最浅，页面底 */
  --paper-1: #f3ece0;   /* 卡片底 */
  --paper-2: #ebe2d2;   /* 输入/分隔 */
  --paper-edge: #e0d5c2; /* 描边 */

  /* 墨色（文字/线条） */
  --ink-900: #2b2620;  /* 标题、正文主色 */
  --ink-700: #4a4339;  /* 次级文字 */
  --ink-500: #7a7062;  /* 辅助说明 */
  --ink-300: #a89e8d;  /* 占位、禁用 */

  /* 朱砂（主强调色，行动召唤） */
  --cinnabar-600: #c0392b;
  --cinnabar-500: #d94f3d;
  --cinnabar-100: #f7e3df;  /* 浅底高亮 */

  /* 青黛（次强调，链接/信息） */
  --indigo-600: #3a5a78;
  --indigo-100: #e3ebf2;

  /* 黛绿（成功/已掌握） */
  --jade-600: #3a8f6f;
  --jade-100: #def0e7;

  /* 藤黄（提醒/待复习） */
  --amber-600: #c8881f;
  --amber-100: #f6ecd4;

  /* 功能色 */
  --success: var(--jade-600);
  --warning: var(--amber-600);
  --danger:  var(--cinnabar-600);
}
```

**对比度**：`--ink-900` on `--paper-0` ≈ 11:1（远超 AA）；`--cinnabar-600` on `--paper-0` ≈ 4.8:1（AA 通过）。

### 4.2 字体 Tokens

```css
:root {
  --font-zh-display: 'Noto Serif SC', 'Songti SC', serif;  /* 诗词正文/标题：宋体 */
  --font-zh-ui:      'Noto Sans SC', 'PingFang SC', sans-serif; /* UI 文案：黑体 */
  --font-pinyin:     'Noto Sans SC', sans-serif;           /* 拼音 */
  --font-num:        'Baloo 2', 'Noto Sans SC', sans-serif; /* 数字：圆润童趣 */
}
```

字号阶梯（rem，根 16px）：

| Token | 大小 | 用途 |
|-------|------|------|
| `--fs-poem` | 2.0rem (32px) | 诗词正文单字 |
| `--fs-h1` | 1.75rem | 页面主标题 |
| `--fs-h2` | 1.375rem | 区块标题 |
| `--fs-body` | 1.0625rem (17px) | 正文（比默认略大，儿童友好） |
| `--fs-meta` | 0.9375rem | 元信息 |
| `--fs-pinyin` | 0.875rem | 拼音 |

### 4.3 间距 / 圆角 / 阴影 / 动效

```css
:root {
  /* 8pt 栅格 */
  --s-1:4px; --s-2:8px; --s-3:12px; --s-4:16px; --s-5:24px;
  --s-6:32px; --s-7:48px; --s-8:64px;

  /* 圆角：偏大、柔和、童趣 */
  --r-sm:8px; --r-md:14px; --r-lg:20px; --r-pill:999px;

  /* 阴影：暖色、轻柔 */
  --shadow-sm: 0 1px 3px rgba(80,60,30,0.08);
  --shadow-md: 0 6px 20px rgba(80,60,30,0.10);
  --shadow-lg: 0 16px 40px rgba(80,60,30,0.14);

  /* 动效 */
  --ease-out: cubic-bezier(0.16,1,0.3,1);
  --ease-spring: cubic-bezier(0.34,1.56,0.64,1);
  --t-fast:120ms; --t-base:220ms; --t-slow:420ms;
}
@media (prefers-reduced-motion: reduce) {
  * { animation-duration:0.01ms !important; transition-duration:0.01ms !important; }
}
```

### 4.4 标志性元素

- **宣纸纹理**：页面底用极淡的纸纤维噪点（CSS `background-image` 渐变叠加，无图片依赖）。
- **朱砂印章**：品牌「诗」字采用红底白字方形印章样式，圆角 `--r-md`。
- **墨线分隔**：区块间用渐隐的水墨横线（`linear-gradient` 两端透明）。

## 5. 组件库规范

所有组件定义在 `main.css`，JS 仅输出结构。命名采用 BEM。

### 5.1 按钮 `.btn`

| 变体 | class | 用途 |
|------|-------|------|
| 主行动 | `.btn--primary` | 朱砂实底，白字（开始复习、去考核） |
| 次行动 | `.btn--secondary` | 描边，墨字 |
| 幽灵 | `.btn--ghost` | 透明，hover 浅底 |
| 危险 | `.btn--danger` | 删除用户等 |
| 尺寸 | `.btn--sm` / `.btn--lg` | |
| 圆形图标 | `.btn--icon` | 仅图标，`aria-label` 必填 |

状态：`:hover` 上浮 1px + 阴影加深；`:active` 下压；`:disabled` 降透明度并 `cursor:not-allowed`；`:focus-visible` 朱砂描边环。

### 5.2 卡片 `.card`

- `.card`：基础卡，`--paper-1` 底、`--r-lg` 圆角、`--shadow-md`。
- `.card--interactive`：可点，hover 上浮。
- `.card--accent`：左侧朱砂色条，用于「今日待复习」横幅。

### 5.3 统计卡 `.stat-card`

替代首页内联样式。结构：标签 + 大数字 + 进度条。数字用 `--font-num`，进度条用对应功能色。

### 5.4 徽章 `.badge`

学习状态：`.badge--new`（未学，灰）、`.badge--learning`（学习中，青黛）、`.badge--review`（待复习，藤黄）、`.badge--mastered`（已掌握，黛绿）。

### 5.5 表单

- `.input`：搜索框、文本框，浅底圆角，`:focus` 朱砂描边。
- `.select`：下拉，统一箭头图标。
- `.chip`：可点标签（朝代/作者快捷筛选、考核选项）。

### 5.6 反馈

- `.toast`：右上角浮层，2.5s 自动消失（收藏成功、切换用户等）。
- `.skeleton`：骨架屏（加载诗词列表/详情时）。
- `.empty-state`：空状态，含插画占位 + 文案 + 可选行动按钮。

## 6. 页面设计

### 6.1 整体布局

```
┌───────────────────────────────────────────────┐
│  侧边栏(桌面) / 底部 Tab(移动)                   │
│  ┌──────┐  ┌─────────────────────────────────┐ │
│  │ 印章 │  │  顶部问候条（页面标题 + 用户）    │ │
│  │ 诗   │  ├─────────────────────────────────┤ │
│  │      │  │                                 │ │
│  │ 首页 │  │        主内容区 #app-main         │ │
│  │ 学习 │  │     （max-width 容器 + 留白）     │ │
│  │ 复习 │  │                                 │ │
│  │ 考核 │  │                                 │ │
│  │ 进度 │  │                                 │ │
│  │      │  │                                 │ │
│  │ 用户 │  │                                 │ │
│  └──────┘  └─────────────────────────────────┘ │
└───────────────────────────────────────────────┘
```

- 桌面（≥1024px）：左侧固定竖向导航（宽 88px，图标 + 文字）。
- 平板（768–1023px）：左侧导航收窄或顶部。
- 移动（<768px）：底部 Tab Bar（5 项 + 用户）。

### 6.2 首页 Home

去除赛博文案，改为温暖问候 + 今日任务 + 学习概览 + 快捷入口。

- **问候条**：「早上好，小明！」+ 日期农历感（可选）。
- **今日待复习横幅** `.card--accent`：大数字（圆体）+ 鼓励文案 + 「开始复习」主按钮；为 0 时显示「今天复习完成啦 🎉」。
- **学习概览**：3 张 `.stat-card`（已学 / 已掌握 / 待复习），含进度条。
- **快捷入口**：4 个大图标卡（学新诗 / 复习 / 闯关考核 / 我的进度）。
- **继续上次**（可选）：最近学习的诗词卡片，一键回到详情。

### 6.3 诗词详情页 Poem Detail（重点重做）

布局自上而下：

1. **返回**（左上角 `.btn--icon`）。
2. **配图区**：有图显示图；无图显示宣纸占位 +「✨ 配图生成中」。
3. **标题区**：诗题（大宋体）+ 朝代·作者·体裁（朱砂小标签）+ 收藏星（右上角，独立常驻）。
4. **正文区**：每句一行，**逐字大字** + 上方拼音；当前朗读字高亮（卡拉 OK 式，若用 TTS 按句高亮）。
5. **朗读控件 `.audio-bar`**（重新设计）：

```
┌──────────────────────────────────────────────┐
│   ( ▶ 播放 )   速度: [常速][慢速]   🔖 收藏     │
└──────────────────────────────────────────────┘
```

**音频状态机**（修复死按钮的核心）：

```
              ┌──────────[播放结束/停止]──────────┐
              ▼                                   │
        [idle 待播放] ──[点播放]──▶ [playing 朗读中] ──[点暂停]──▶ [paused 暂停]
              ▲                                                      │
              └──────────────────[点播放]────────────────────────────┘
```

- 主按钮在 `idle/paused` 显示「▶ 播放」，在 `playing` 显示「⏸ 暂停」。
- 速度切换 `[常速][慢速]` 为分段控件，实时生效（修改 `playbackRate` 或 TTS `rate`）。
- **音源优先级**：
  1. `poem.audio`（AI 生成的 base64）→ 用 `HTMLAudioElement`；
  2. 无 `poem.audio` → 用浏览器 `speechSynthesis` 朗读 `poem.content`（中文语音、可调速）；
  3. 都不可用（浏览器不支持）→ 按钮显示「朗读暂不可用」并禁用，附说明。
- **收藏**：独立常驻按钮，点击即存 `localStorage` 并弹 `.toast`「已收藏 ⭐ / 已取消」。

6. **释义区块**：创作背景 / 字词注释 / 主题思想 / 白话翻译，统一 `.card` + 折叠面板（默认展开背景，其余可折叠，减轻儿童认知负担）。
7. **底部行动**：「我学完了，去闯关 →」主按钮。

### 6.4 学习列表 Learn List

- 顶部：标题 +「共 N 首」+ 搜索框。
- 筛选：年级 / 朝代 / 作者下拉 **改为横向 `.chip` 快捷筛选**（更适合触摸，点选高亮）。
- 列表：诗词卡片网格（缩略图/印章占位 + 题名 + 朝代作者 + 状态徽章）。
- 空状态：`.empty-state`「没有找到相关诗词，换个词试试～」。
- 骨架屏：首次加载显示 6 张 `.skeleton` 卡。

### 6.5 复习 Review

- 进入显示今日待复习清单 + 「开始复习」。
- 复习进行：顶部进度条（第 m / n 首）+ 当前题（默认填空，混合题型）。
- 完成：复习报告卡（正确率、用时、鼓励语 + 表情），「再来一组 / 返回首页」。
- 待复习为 0：庆祝空状态。

### 6.6 考核 Quiz（闯关）

四种题型统一外壳 `.quiz-shell`：题干区 + 作答区 + 底部「提交 / 下一题」。

- **填空**：诗句挖空 + 字卡选择池（点字填空，可撤销）。
- **选择**：题干 + 4 个 `.chip` 选项，选中高亮，提交后对错染色。
- **排序**：诗句乱序卡片，拖拽或点选排序。
- **听诗选诗**：播放音频/TTS + 4 个候选。
- 计分反馈：每题即时对错动效（✓ 黛绿 / ✗ 朱砂 + 轻微抖动），结算页给星级（1–3 星）。
- 题型入口要清晰，去掉直接崩溃的边界（无 `keySentences` 时跳过填空或友好提示）。

### 6.7 进度 Progress

- 顶部用户卡（头像 + 名字 + 年级，点击切换/管理用户）。
- 总览：环形/条形进度（已学占比、已掌握占比）。
- 朝代分布：小型饼图（已有 `renderDynastyPieSvg`，重新配色）。
- 成就墙：成就徽章（已解锁高亮，未解锁灰显 + 进度）。
- 多用户管理：新增 / 重命名 / 重置 / 删除（带二次确认）。

### 6.8 打印 Print（家长用）

- 四种版式（描红 / 默写 / 紧凑 / 讲义）预览卡，选择后 `window.print()`。
- 打印样式 `print.css` 同步换成浅色高对比、省墨排版。

## 7. 响应式与无障碍

### 7.1 断点

| 断点 | 宽度 | 布局 |
|------|------|------|
| 移动 | <768px | 底部 Tab，单列 |
| 平板 | 768–1023px | 侧栏 + 单/双列 |
| 桌面 | ≥1024px | 侧栏 + 多列网格，内容 `max-width:1100px` 居中 |

### 7.2 无障碍清单

- 所有交互元素可 `Tab` 聚焦，`:focus-visible` 有可见焦点环。
- 图标按钮必带 `aria-label`；音频状态用 `aria-pressed` / `aria-live` 播报。
- 颜色不作为唯一信息载体（对错同时用图标）。
- 对比度 ≥ AA；正文字号 ≥17px。
- 尊重 `prefers-reduced-motion`。

### 8.3 音频兜底（修复死按钮）

```js
// audio.js 新增
export function createSpeech(lines, { rate = 1 } = {}) {
  const text = Array.isArray(lines) ? lines.join('，') : String(lines);
  const u = new SpeechSynthesisUtterance(text);
  u.lang = 'zh-CN';
  u.rate = rate;
  return u;
}
export function speak(utt) { speechSynthesis.cancel(); speechSynthesis.speak(utt); }
export function stopSpeak() { speechSynthesis.cancel(); }
export function speechSupported() { return 'speechSynthesis' in window; }
```
