# 诗云 · 古诗词学习管理系统 · 设计文档

> **项目代号：** 诗云 (ShiYun)
> **创建日期：** 2026-06-16
> **目标用户：** 家庭（家长 + 1-2 个 1-6 年级小学生）
> **核心功能：** 学习 / 复习 / 考核 / 打印 / 学习进度管理
> **诗词范围：** 部编版语文 1-6 年级 112 首古诗词

---

## 1. 项目概述

### 1.1 愿景
打造"**双击即用 + 公众号生态 + 微信小游戏**"三位一体的家庭古诗词学习平台：

- **离线版（核心）**：家长用 AI 一次性生成所有内容，孩子日常使用完全离线、学习进度自动追踪
- **公众号生态**：把公众号作为系统的分发渠道、内容发布平台、学习分享渠道
- **微信小游戏**：通过游戏化（闯关/飞花令/拼诗）让学习更有趣、扩大影响力

### 1.2 核心价值
- **零运维**：纯静态 HTML，浏览器双击打开
- **AI 个性化**：112 首诗词全部由 AI 配图、配音、注释，贴合诗词意境
- **科学复习**：基于 SM-2 间隔重复算法，让孩子在记忆临界点复习
- **多维筛选**：按年级、作者、朝代、复习需求筛选
- **A4 打印**：生成可随身携带的复习卡片
- **公众号生态**：分发 + 内容运营 + 分享裂变
- **小游戏化**：闯关 + 对战 + 拼图，三种游戏模式

### 1.3 不做什么（YAGNI）
- ❌ 用户注册、登录系统（家庭单用户，localStorage 足够；小游戏用微信一键登录）
- ❌ 多设备同步（学习版可复制到多设备，各自独立）
- ❌ 服务端 / 数据库（除小游戏排行榜/飞花令对战外，不引入后端）
- ❌ 移动 App（响应式 Web 已覆盖手机/平板）
- ❌ 朗读评分 / 语音识别（避免 ASR 复杂度）
- ❌ 支付 / 商业化（首版免费）
- ❌ 服务号（仅支持个人订阅号）

**数据流：**
1. 生成器 AI 生成内容 → 嵌入到学习版 HTML
2. 学习版嵌入的数据可被公众号生态和小游戏引用（如配图、原文）
3. 公众号生态分发学习版 + 发布内容文章
4. 小游戏独立运行（共享诗词数据库 JSON）

### 2.2 技术栈
| 类别 | 选型 | 理由 |
|------|------|------|
| 前端框架 | 原生 HTML/CSS/JS | 无构建、双击即用、单文件分发 |
| 数据存储 | localStorage | 学习进度小、无后端 |
| 媒体存储 | base64 内嵌 HTML | 单文件可分发 |
| AI 服务 | OpenAI 一站式（默认），架构支持接入其他 provider | 文本 (GPT-4o-mini) + 图像 (DALL-E 3) + 语音 (TTS-1) |
| 拼音库 | pinyin-pro | 浏览器权威方案 |
| 打印 | 原生 `window.print()` + `@media print` | 零依赖 |
| 构建工具 | Python 脚本 (`build-learning.py`) | 简单可靠、无 Node 依赖 |

### 2.3 依赖项
- 现代浏览器（Chrome 90+ / Edge 90+ / Firefox 88+）
- 生成阶段需要联网 + OpenAI API Key
- 学习阶段完全离线

## 3. 数据模型

### 3.1 诗词库（嵌入 HTML，poems.json）

```typescript
interface Poem {
  id: string;              // "g1-jingyesi"（年级+拼音简称）
  title: string;           // "静夜思"
  author: string;          // "李白"
  dynasty: string;         // "唐"
  grade: 1 | 2 | 3 | 4 | 5 | 6;
  type: string;            // "五言绝句"
  content: string[];       // ["床前明月光", "疑是地上霜", ...]
  pinyin: string[];        // ["chuáng qián míng yuè guāng", ...]

  // AI 生成内容
  translation: string;     // 白话翻译
  background: string;      // 创作背景
  annotations: { [char: string]: string };  // 字词释义
  theme: string;           // 主题思想
  keywords: string[];      // 主题关键词

  // 媒体资源（base64）
  image: string;           // "data:image/jpeg;base64,..."
  audio: string;           // "data:audio/mp3;base64,..."

  // 考核用关键句
  keySentences: Array<{
    line: number;          // 第几句
    chars: string[];       // 拆开的字
    blanks: number[];      // 挖空位置（索引）
  }>;
}
```

### 3.2 学习进度（localStorage）

```typescript
interface UserState {
  version: string;
  currentUser: string;
  users: {
    [userId: string]: {
      name: string;
      avatar: string;         // emoji
      grade: 1 | 2 | 3 | 4 | 5 | 6;
      createdAt: string;
      poemProgress: {
        [poemId: string]: {
          status: 'new' | 'learning' | 'reviewing' | 'mastered';
          learnCount: number;     // 学习次数
          quizCount: number;      // 考核次数
          avgScore: number;       // 平均分（0-100）
          lastLearnedAt: string;  // ISO date
          nextReviewAt: string;   // SM-2 算法计算的下次复习日
          easeFactor: number;     // SM-2 ease factor
          favorite: boolean;
          notes: string;
        };
      };
      quizHistory: Array<{
        poemId: string;
        mode: 'fill' | 'choice' | 'order' | 'listen';
        score: number;
        at: string;
      }>;
      achievements: string[];   // ['first-poem', 'ten-mastered', ...]
    };
  };
}
```

### 3.3 SM-2 间隔重复算法（简化版）

```
考核得分 ≥ 90  →  interval = interval × 2.5（已掌握）
考核得分 70-89 →  interval = interval × 1.5（待巩固）
考核得分 < 70  →  interval = 1（重置）
连续 3 次 ≥ 90 →  status = 'mastered'

"今日复习"列表：所有 nextReviewAt ≤ today 的诗
```

### 3.4 状态机

```
new (未学) ──[开始学习]──→ learning ──[3天内]──→ reviewing ──[连续3次≥90分]──→ mastered
                              │                       │
                              └────[得分<70]───────────┘
                              (回到 learning)
```

## 4. 五大功能模块

### 4.1 📚 学习（Learn）

**入口路径：** 首页 → 学新诗 → 诗词列表 → 单首详情

**诗词列表页：**
- 顶部筛选：年级（1-6）、朝代（唐/宋/...）、作者、有图/无图
- 列表项：缩略图 + 标题 + 作者朝代 + 状态徽章（new / learning / 已学 X 次 / 已掌握 ⭐）
- 支持搜索（按标题、作者）

**单首详情页：**
- 顶部：AI 配图（视觉锚定）
- 中部：标题、作者朝代、原文（楷体大字号）、拼音（小字号）
- 操作按钮：▶ 朗读 / 慢速 / 跟读 / 收藏
- 下方：创作背景、字词释义、主题思想
- 底部：「我学完了」按钮（进入考核）

### 4.2 🔄 复习（Review）

**入口路径：** 首页 → 今日复习（待复习 N 首）

**自动筛选：**
- SM-2 算法：所有 `nextReviewAt ≤ 今天` 的诗
- 显示"今日待复习"卡片（数量 + 行动按钮）
- 支持手动模式：选年级、选状态筛选

**复习流程：**
- 直接进入考核（填空模式优先）
- 考核后自动更新 SM-2 参数

### 4.3 ✏️ 考核（Quiz）

**4 种模式：**

| 模式 | 难度 | 适用 | 实现 |
|------|------|------|------|
| A. 诗句填空 | ★★ | 全年级 | 挖掉 1-2 个字，孩子从字库选填 |
| B. 选择题 | ★ | 低年级 | 给句选下句 / 给作者选作品（4选1） |
| C. 诗句排序 | ★★★ | 中高年级 | 打乱 4 句，孩子拖拽排序 |
| D. 听诗选诗 | ★★ | 全年级 | 播放 AI 朗读，4 首诗名里选正确 |

**考核配置：**
- 模式选择（单选或混合）
- 范围（年级 / 朝代 / 状态）
- 题目数量（10 / 20 / 50 / 全部）
- 完成后显示得分、错题回顾、正确答案

### 4.4 🖨️ 打印（Print）

**入口路径：** 首页 → 打印 → 筛选 → 版式选择 → 浏览器打印（Ctrl+P）

**版式（默认：密集复习卡）：**
- **密集复习卡**（默认）：一页 4-6 首 + 完整教学讲义（背景、注释、主题），可随身携带
- 经典欣赏版：单页一首 + 配图 + 拼音
- 默写练习版：拼音填字 + 理解题
- 学习讲义版：完整教学讲义

**筛选条件：**
- 按年级 / 朝代 / 作者
- 按复习需求：今日待复习 / 全部已学 / 已收藏
- 按关键词

**打印输出：**
- A4 纸张，纵向/横向可选
- 浏览器原生打印对话框 → 另存为 PDF / 直连打印机

### 4.5 📊 进度管理（Progress）

**入口路径：** 首页 → 进度

**统计概览：**
- 已学 / 已掌握 / 待复习 / 总数
- 学习曲线（最近 30 天）
- 朝代分布饼图
- 成就徽章墙

**个人设置：**
- 切换用户（1-2 个孩子切换）
- 修改姓名 / 头像 / 年级
- 重置进度（带确认）
- 导出/导入进度（JSON 文件）

## 5. 生成器设计（家长用·一次性）

### 5.1 工作流

```
[步骤 1] 配置 API Key  →  [步骤 2] 选择年级范围  →  [步骤 3] AI 生成  →  [步骤 4] 导出
```

### 5.2 每首诗的生成内容

| 内容 | AI 模型（默认 OpenAI） | 提示词要点 | 规格 |
|------|---------|-----------|------|
| 翻译 | GPT-4o-mini | 通俗易懂、孩子能理解 | < 100 字 |
| 背景 | GPT-4o-mini | 历史背景、创作故事 | < 150 字 |
| 注释 | GPT-4o-mini | 难字难词、字面+引申 | 3-5 个 |
| 主题 | GPT-4o-mini | 思想感情、艺术特色 | < 100 字 |
| 关键词 | GPT-4o-mini | 主题词、意象 | 3-5 个 |
| 关键句挖空 | GPT-4o-mini | 选最有代表性的 2-3 个字 | 结构化 JSON |
| 配图 | DALL-E 3 | 传统水墨 + 诗词主题 | 512×512 JPEG |
| 朗读 | TTS-1 | alloy/shimmer voice，节奏慢、清晰 | MP3 64kbps |

### 5.3 断点续传与可靠性

- 每首生成完立即写入 IndexedDB / localStorage
- 中途关闭可恢复（按 ID 跳过已生成）
- 单首失败不影响其他，可单独重生成
- 实时预览某首生成效果

### 5.4 内置诗词清单（112 首）

生成器内置完整的部编版 1-6 年级 112 首诗词的元数据（标题、作者、朝代、年级、原文），家长无需手动输入。AI 只需生成"扩展内容"（翻译、背景、配图、音频等）。

## 9. 错误处理与边界

| 场景 | 处理 |
|------|------|
| API Key 无效 | 生成器实时校验，红色提示 |
| AI 生成某首失败 | 单首标记失败、可重试，不阻塞其他 |
| 网络中断 | 断点续传，下次打开恢复 |
| 浏览器不兼容 | 在生成器首页检测，提示升级 |
| localStorage 满 | 提示用户导出进度，重置 |
| 误删学习版 | 从备份（家长生成的源文件）重新生成 |
| 误操作清除进度 | 设置页"导入进度"JSON 文件恢复 |

## 11. 公众号生态（诗云-公众号）

公众号生态包含 4 大功能，让公众号成为 诗云 的分发、内容运营、分享裂变渠道。

### 11.1 公众号菜单分发（核心入口）

**菜单设计：**
```
┌─ 诗云 ──────────────┐    ┌─ 学习 ───────────┐    ┌─ 更多 ───────────┐
│ 📖 开始学习         │ →  │ ▶ 离线版下载      │    │ 📰 往期内容      │
│ 🎮 进入小游戏       │    │ ❓ 学习指南        │    │ 📤 我的海报      │
│ 📤 分享给好友       │    │ 💬 问题反馈        │    │ ⭐ 关于诗云      │
└─────────────────────┘    └───────────────────┘    └───────────────────┘
```

**实现方式：**
- "📖 开始学习"：跳转到学习版 HTML 的云存储链接（对象存储 OSS/COS + CDN）
- "🎮 进入小游戏"：跳转到小游戏
- "📤 分享给好友"：触发微信 JS-SDK 分享接口
- "❓ 学习指南"：公众号文章
- "📰 往期内容"：公众号文章列表

**更新流程：**
- 家长重新生成学习版后 → 上传新文件到 OSS → 公众号菜单链接不变（指向 OSS 路径）

### 11.2 自动发布内容文章

**自动化工作流：**
```
[生成器生成完毕]
  ↓
[一键推送：自动创建公众号草稿]
  ↓
[云函数调用公众号 API：新增草稿接口]
  ↓
[你登录公众号后台审核 + 群发]
  ↓
[粉丝看到本周新诗文章]
```

**文章模板：**
- 📰 "诗云·本周新诗推荐"：3-5 首本周新增诗词，配图、原文、一句赏析
- 📊 "诗云·每月学习报告"：本月孩子学习数据统计（需孩子授权推送）
- 🎯 "诗云·学习技巧"：教育性内容

**技术实现：**
- 需要公众号 AppID + AppSecret（IP 白名单）
- 微信云函数（推荐）或自有服务器调用 API
- 调 `draft/add` 接口创建草稿，调用 `freepublish/submit` 群发

**频率限制：**
- 个人订阅号：1 次/天群发
- 草稿数量无限制

### 11.3 学习海报分享

**触发场景：**
- 学习版里"我的进度"页 → "生成我的学习海报"
- 海报内容：孩子的头像、本周学习数、掌握数、连续学习天数、Top 3 诗词

2. `html2canvas` 把 HTML 节点转图片
3. 微信 JS-SDK `updateAppMessageShareData` 配置分享
4. 用户点击微信"分享到朋友圈"

**示例海报结构：**
```

│  ───────────────────────   │
│  🐯 小明 · 三年级             │
│  本周学习了 5 首诗            │
│  累计掌握 12 首               │
│  🔥 连续学习 7 天             │
│                              │
│  本周 Top 3：                 │
│  ⭐ 静夜思 · 春晓 · 登鹳雀楼   │
│                              │
│  [诗云二维码]                 │
│  扫码体验古诗词学习           │
└─────────────────────────────┘
```

### 11.4 关注引导

**位置：**

## 12. 微信小游戏（诗云-小游戏）

小游戏是 诗云 学习版的"社交化 + 游戏化"补充，让孩子在玩中复习、扩大影响力。

### 12.1 三种游戏模式

| 模式 | 玩法 | 实现难度 | 周期 |
|------|------|----------|------|
| 🏆 答题闯关 | 单人闯关 / 双人对战，按关卡答题（每关 10 题） | ★★ | 2 周 |
| 🧩 拼诗游戏 | 拖动字块组成完整诗句 | ★★ | 2 周 |
| 🎴 飞花令 | 给关键词，双方轮流说含此字的诗句（多人对战） | ★★★ | 3 周 |

### 12.2 共享数据

**诗词数据复用：**
- 从学习版 `poems-meta.js` 导出精简版 `poems-mini.json`
- 字段：id, title, author, dynasty, content, pinyin, image, audio
- 小游戏首屏加载（约 5-10 MB），后续按需加载

**音频处理：**
- 小游戏对包大小敏感（初始包 4MB 限制）
- 音频改为按需加载（点击播放时才下载）
- 或使用微信 TTS API（`wx.createInnerAudioContext` + 远程 URL）

### 12.3 闯关模式详细设计

**关卡设计：**
- 第 1-10 关：1 年级（按教材顺序）
- 第 11-25 关：2 年级
- 第 26-50 关：3 年级
- 第 51-80 关：4 年级
- 第 81-105 关：5 年级
- 第 106-112 关：6 年级（终极关卡）

**每关结构：**
- 10 道题（混合 4 种题型：填空/选择/排序/听诗选诗）
- 错题可重答，全对解锁下一关
- 通关奖励：成就徽章 + 解锁诗词背景故事

**画面：**
- 顶部：关卡地图（古风长卷轴风格，112 个节点）
- 中部：当前题目
- 底部：选项区

### 12.4 飞花令模式详细设计

**核心逻辑：**
- 玩家 A 给关键词（如"月"），玩家 B 说出含"月"的诗句
- 10 秒内回答，AI 判断诗句是否正确
- 答不出/答错则对方得 1 分

**AI 判题实现：**
- 接入 OpenAI API（GPT-4o-mini）：判断诗句是否含关键词、是否正确
- 客户端缓存常见诗句（避免每次调用 API）

**对战匹配：**
- 好友邀请（微信好友）
- 随机匹配（同水平段位）
- 段位系统：青铜 → 白银 → 黄金 → 铂金 → 钻石

**技术实现：**
- WebSocket 服务端（需要一个小后端）
- 微信鉴权（`wx.login()` + 后端校验）

### 12.5 拼诗游戏详细设计

**玩法：**
- 屏幕上散落字块，拖到正确位置
- 简单关卡：3 选 1
- 中等关卡：5 字拼 5 句
- 困难关卡：跨多句混排

**技术：**
- 微信小游戏 Canvas API
- 触摸事件适配
- 字块使用预渲染（性能）

### 12.6 微信生态能力集成

| 能力 | 用途 | 接入方式 |
|------|------|----------|
| `wx.login()` | 用户身份识别 | 一键登录 |
| `wx.getUserInfo()` | 头像/昵称 | 默认头像 |
| `wx.shareAppMessage()` | 好友分享 | 邀请对战 |
| `wx.onShareTimeline()` | 朋友圈分享 | 炫耀成绩 |
| `wx.createBannerAd()` | 激励视频广告 | 复活/提示（可选） |
| `wx.cloud.storage` | 云端存档 | 跨设备进度同步 |
| `wx.openCustomerServiceConversation()` | 客服 | 用户反馈 |

### 12.7 小游戏提审

**前置条件：**
- 微信小程序（小游戏）账号注册
- 主体认证（个人 / 企业）
- AppID 申请
- 游戏类目资质（个人小游戏需提供软件著作权或选择文教类目）

**审核要点：**
- 内容合规（无暴力/低俗）
- 用户隐私（个人信息收集声明）
- 支付合规（首版不接入支付，规避）
- 性能（首屏 4s 内打开）

**上线流程：**
1. 微信开发者工具上传代码
2. 填写小程序信息
3. 提交审核（1-3 天）
4. 审核通过 → 发布

## 附录 A：112 首诗词清单（内容数据源）

**数据来源：** 教育部《义务教育语文课程标准》推荐 + 部编版（人教社）小学语文 1-6 年级教材必背篇目，共 112 首。

**典型分布参考：**
- 1 年级上：8 首（如《静夜思》《咏鹅》《江南》《画》等）
- 1 年级下：4 首
- 2 年级上：6 首 / 2 年级下：8 首
- 3 年级上：10 首 / 3 年级下：10 首
- 4-6 年级：66 首

**生成器需要的数据：**

```typescript
interface PoemMeta {
  id: string;        // "g1-jingyesi"（年级-002序号）
  title: string;     // 诗词标题
  author: string;    // 作者
  dynasty: string;   // 朝代
  grade: 1|2|3|4|5|6;
  type: string;      // "五言绝句"/"七言绝句"/"五言律诗"/"七言律诗"/"词"/"其他"
  content: string[]; // 分句数组（去掉标点）
  sequence: number;  // 教材中的顺序
}
```

**实施要求（创建 `src/data/poems-meta.js` 时执行）：**
1. 严格按部编版教材官方版本整理 112 首
2. 包含完整作者、朝代、原文
3. 每首标注在教材中的年级与顺序
4. 提供 JSON Schema 校验脚本确保无遗漏
5. 内容来源标注（如"部编版 1 年级上 P8"）

> 实施时此文件为"内容数据"，不属于设计决策；具体 112 首清单在实施阶段以"内容数据源"任务独立完成。
